CHƯƠNG 3. PHÁT TRIỂN HỆ THỐNG
3.1. XÂY DỰNG GIAO DIỆN VÀ CƠ SỞ DỮ LIỆU

3.1.1. GIỚI THIỆU

Giai đoạn xây dựng giao diện và cơ sở dữ liệu là bước quan trọng nhất để chuyển 
từ thiết kế lý thuyết sang sản phẩm thực tế hoạt động. Phần này mô tả chi tiết quá 
trình triển khai cả hai khía cạnh: phía sau lưng (Backend - Cơ sở dữ liệu) và phía 
trước (Frontend - Giao diện người dùng) cho website Tin tức & Blog.

Dựa trên kế hoạch đã nêu trong Giai đoạn 2 (Phát triển) của dự án, nhóm đã tiến hành:
  • Xây dựng mô hình dữ liệu (Database Models) bằng Django ORM
  • Khởi tạo cơ sở dữ liệu SQLite3 và tạo các bảng dữ liệu
  • Thiết kế giao diện người dùng (Frontend) với HTML, CSS, JavaScript
  • Tích hợp template engine Django vào các trang web

───────────────────────────────────────────────────────────────────────────────

3.1.2. XÂY DỰNG CƠ SỞ DỮ LIỆU

3.1.2.1. KIẾN TRÚC CƠ SỞ DỮ LIỆU

Theo phương án đề xuất trong Chương 1, nhóm sử dụng SQLite3 làm hệ quản trị cơ sở 
dữ liệu chính bởi những ưu điểm sau:

  ✓ Nhẹ và tích hợp sẵn: SQLite3 được tích hợp mặc định trong Django, không cần 
    cài đặt server riêng

  ✓ Phù hợp giai đoạn phát triển: Hoàn toàn đủ cho ứng dụng blog và tin tức quy 
    mô vừa (vài trăm bài viết, vài nghìn lượt truy cập/ngày)

  ✓ Dễ sao lưu và triển khai: Cơ sở dữ liệu lưu trữ trong file duy nhất db.sqlite3, 
    dễ backup và di chuyển

  ✓ Nền tảng dự phòng: Khi mở rộng, hệ thống có thể nâng cấp lên PostgreSQL hoặc 
    MySQL mà không cần thay đổi code Django (nhờ Django ORM)

3.1.2.2. THIẾT KẾ CÁC MODEL (BẢNG DỮ LIỆU)

Dựa trên thiết kế cơ sở dữ liệu ở Chương 2.3, nhóm đã xây dựng các Django Model 
sau:

A) MODEL USER (NGƯỜI DÙNG)
   Thuộc tính:
   • user_id: Khóa chính (Primary Key), tự động tăng
   • username: Tên đăng nhập (Unique)
   • password: Mật khẩu (được hash bằng Django)
   • full_name: Họ tên đầy đủ
   • email: Địa chỉ email
   • role_id: ID vai trò (Admin, Editor, Reader)
   • created_at: Ngày tạo tài khoản
   • is_active: Trạng thái kích hoạt tài khoản

   Mô tả: Lưu trữ thông tin đăng nhập và hồ sơ của tất cả người dùng trên hệ thống 
   (Quản trị viên, Biên tập viên, Độc giả).

B) MODEL CATEGORY (DANH MỤC)
   Thuộc tính:
   • category_id: Khóa chính
   • name: Tên danh mục (Unique)
   • slug: URL-friendly identifier (Unique), tự động sinh từ name
   • description: Mô tả danh mục
   • category_type: Loại danh mục ('news' hoặc 'blog')
   • created_at: Ngày tạo danh mục

   Mô tả: Phân loại các bài viết thành Tin tức và Blog. Mỗi danh mục có thể chứa 
   nhiều bài viết con.

C) MODEL TAG (THẺ TỪ KHÓA)
   Thuộc tính:
   • tag_id: Khóa chính
   • name: Tên thẻ (Unique)
   • slug: URL-friendly identifier (Unique)

   Mô tả: Hệ thống từ khóa gắn liền với bài viết, cho phép tìm kiếm và liên kết 
   nội dung nhanh chóng.

D) MODEL POST (BÀI VIẾT)
   Thuộc tính:
   • post_id: Khóa chính
   • title: Tiêu đề bài viết
   • slug: URL-friendly identifier (Unique), tự động sinh từ title
   • content: Nội dung bài viết (TextField - hỗ trợ HTML)
   • summary: Tóm tắt ngắn gọn (được hiển thị trong danh sách)
   • thumbnail: Đường dẫn hình ảnh đại diện
   • category_id: Khóa ngoài tham chiếu Category
   • author_id: Khóa ngoài tham chiếu User
   • views_count: Số lượt xem
   • status: Trạng thái ('draft' - nháp, 'published' - đã đăng)
   • post_type: Loại bài viết ('news' - tin tức, 'blog' - blog)
   • created_at: Ngày tạo
   • updated_at: Ngày cập nhật lần cuối
   • published_at: Ngày xuất bản (tự động ghi lại khi chuyển sang 'published')

   Mô tả: Lưu trữ nội dung bài viết tin tức và blog. Mỗi bài viết liên kết với một 
   danh mục, một tác giả, và có thể gắn nhiều thẻ.

E) MODEL TAG_POST (MỐI QUAN HỆ NHIỀU-NHIỀU)
   Thuộc tính:
   • post_id (Khóa ngoài tham chiếu Post)
   • tag_id (Khóa ngoài tham chiếu Tag)

   Mô tả: Bảng trung gian lưu trữ mối quan hệ giữa bài viết và các thẻ từ khóa. Cho 
   phép một bài viết gắn nhiều thẻ và một thẻ có thể được sử dụng cho nhiều bài viết.

F) MODEL COMMENT (BÌNH LUẬN)
   Thuộc tính:
   • comment_id: Khóa chính
   • content: Nội dung bình luận
   • post_id: Khóa ngoài tham chiếu Post
   • user_id: Khóa ngoài tham chiếu User
   • parent_id: Khóa ngoài tham chiếu Comment (nếu là reply)
   • is_approved: Boolean - cho biết bình luận đã được duyệt hay chưa
   • created_at: Ngày gửi bình luận

   Mô tả: Lưu trữ bình luận của người dùng trên các bài viết. Hỗ trợ bình luận cấp 
   cha-con (parent-child) cho phép trả lời trực tiếp.

G) MODEL ADVERTISEMENT (QUẢNG CÁO)
   Thuộc tính:
   • ad_id: Khóa chính
   • title: Tiêu đề quảng cáo
   • banner_url: Đường dẫn hình ảnh hoặc URL chuyển hướng
   • position: Vị trí hiển thị ('header', 'sidebar', 'footer')
   • created_at: Ngày tạo quảng cáo

   Mô tả: Quản lý các banner quảng cáo được phân bố ở các vị trí khác nhau trên website.

3.1.2.3. KHỞI TẠO CƠ SỞ DỮ LIỆU

Quá trình khởi tạo cơ sở dữ liệu thực hiện như sau:

BƯỚC 1: TẠO MIGRATION
  $ python manage.py makemigrations Webtintuc
  
  Lệnh này quét tất cả Model đã định nghĩa và tạo file migration (tên như 
  0001_initial.py), mô tả những thay đổi cần thực hiện lên database.

BƯỚC 2: ÁP DỤNG MIGRATION
  $ python manage.py migrate
  
  Lệnh này thực thi các migration, tạo ra các bảng dữ liệu trong file 
  db.sqlite3. Ngoài các bảng của Webtintuc app, Django cũng tự động tạo các 
  bảng cho Django Admin (auth, contenttypes, sessions...).

BƯỚC 3: TẠO SUPER ADMIN
  $ python manage.py createsuperuser
  
  Lệnh này tạo tài khoản quản trị viên cấp cao nhất, cho phép truy cập Django 
  Admin panel để quản lý dữ liệu.

KẾT QUẢ: Cơ sở dữ liệu được khởi tạo hoàn chỉnh với 7 bảng chính (User, 
Category, Tag, Post, Comment, Advertisement, Post_Tag) và các bảng phụ trợ 
của Django.

3.1.3. XÂY DỰNG GIAO DIỆN NGƯỜI DÙNG (FRONTEND)

3.1.3.1. KIẾN TRÚC GIAO DIỆN

Giao diện website được xây dựng theo mô hình MVC (Model-View-Controller) của Django:
  • Model: Đã được mô tả ở mục 3.1.2
  • View: Các hàm Python xử lý logic (lọc dữ liệu, xác thực người dùng...)
  • Template: Các tệp HTML kết hợp CSS/JavaScript để hiển thị giao diện

Tài nguyên tĩnh (Static Files) được tổ chức trong thư mục Webtintuc/static/:

  static/
  ├── css/
  │   ├── style.css        (CSS chính)
  │   └── trang_chu.css    (CSS trang chủ)
  ├── js/
  │   └── trang_chu.js     (JavaScript trang chủ)

3.1.3.2. CẤU TRÚC TEMPLATE

Các Template HTML được tổ chức theo cấu trúc phân cấp:

A) BASE TEMPLATE (base.html)
   
   Template cơ sở chứa:
   • Header: Logo, thanh điều hướng (Mới nhất, Xem nhiều, Tìm kiếm), nút 
     Đăng nhập/Đăng ký
   • Nội dung chính: {% block content %} - phần thay đổi tùy theo từng trang
   • Footer: Liên kết, Điều khoản sử dụng, Chính sách bảo mật, Liên hệ, 
     Social links
   • CSS & JS: Được load một lần duy nhất cho tất cả các trang con

   Mục đích: Tránh lặp lại code, đảm bảo consistency giữa các trang.

B) TEMPLATE TRANG CHỦ (trang_chu.html)

   Cấu trúc:
   
   1. Featured News Section
      - Hiển thị bài viết nổi bật nhất (views cao nhất)
      - Cho phép click để xem chi tiết
   
   2. Latest News Section
      - Danh sách 8-10 bài viết mới nhất
      - Hiển thị: Tiêu đề, Tác giả, Ngày đăng, Tóm tắt
      - Card responsive (thích ứng trên desktop, tablet, mobile)
   
   3. Trending Section
      - Danh sách 5 bài viết có lượt xem cao nhất
      - Sắp xếp giảm dần theo views_count

   Công nghệ sử dụng:
   • Django Template Language (DTL): Dùng vòng lặp {% for %}...{% endfor %} 
     để duyệt qua danh sách bài viết
   • Filters: |date:"d/m/Y" (định dạng ngày), |truncatewords:20 (cắt ngắn 
     đoạn văn)
   • URL Reversal: {% url 'webtintuc:post_detail' post.slug %} - tự động 
     sinh URL an toàn

C) TEMPLATE CHI TIẾT BÀI VIẾT (chi-tiet-tin.html)

   Cấu trúc:
   
   1. Phần Header bài viết
      - Tiêu đề, tác giả, ngày đăng, lượt xem
   
   2. Hình ảnh đại diện (thumbnail)
   
   3. Nội dung chính (content)
      - Hỗ trợ HTML formatting
   
   4. Thẻ từ khóa (Tags)
      - Liên kết click để xem bài cùng tag
   
   5. Bài viết liên quan (Related posts)
      - Hiển thị bài cùng category hoặc cùng tag
   
   6. Phần bình luận
      - Form gửi bình luận (nếu người dùng đã đăng nhập)
      - Danh sách bình luận đã duyệt (is_approved=True)
      - Hỗ trợ trả lời bình luận (nested comments)

D) TEMPLATE DANH MỤC (category_news.html)
   - Hiển thị danh sách bài viết thuộc một danh mục cụ thể
   - Hỗ trợ phân trang (pagination) để tải từng 10 bài viết một

E) TEMPLATE KẾT QUẢ TÌM KIẾM (search_results.html)
   - Hiển thị bài viết khớp với từ khóa người dùng nhập
   - Tìm kiếm được thực hiện trên trường title và content bằng ORM Query

F) TEMPLATE ĐĂNG NHẬP / ĐĂNG KÝ (login.html, register.html)
   - Form nhập username/password hoặc email/mật khẩu
   - Xác thực được thực hiện bằng Django Forms
   - Sau khi đăng nhập thành công, cookie session được lưu trữ

G) TEMPLATE BÀI VIẾT CỦA TÔI (my_posts.html)
   - Hiển thị danh sách bài viết do người dùng hiện tại sáng tác
   - Chỉ truy cập được khi người dùng đã đăng nhập (@login_required decorator)

3.1.3.3. THIẾT KẾ GIAO DIỆN

Ngôn ngữ & Framework:
  • HTML5: Cấu trúc ngữ nghĩa, meta tags responsive
  • CSS3: Flexbox layout, Grid, Media queries cho responsive design
  • JavaScript: Interactivity (animation, form validation, AJAX nếu cần)

Responsive Design:
Website được thiết kế hoạt động mượt mà trên cả:
  • Desktop (≥1024px)
  • Tablet (768px - 1023px)
  • Mobile (< 768px)

Sử dụng CSS Media Queries:

  /* Mobile First */
  .news-grid {
      display: grid;
      grid-template-columns: 1fr;  /* 1 cột trên mobile */
  }
  
  @media (min-width: 768px) {
      .news-grid {
          grid-template-columns: repeat(2, 1fr);  /* 2 cột trên tablet */
      }
  }
  
  @media (min-width: 1024px) {
      .news-grid {
          grid-template-columns: repeat(3, 1fr);  /* 3 cột trên desktop */
      }
  }

Màu sắc & Typography:
  • Color Scheme: Xanh dương (#2c3e50, #3498db) - phù hợp với chủ đề tin tức / 
    công nghệ
  • Font: Segoe UI, Tahoma, Geneva - dễ đọc trên mọi thiết bị
  • Font sizes: Tiêu đề H1 (28px+), nội dung (14-16px), caption (12px)

Layout:
  • Header: Sticky (dính ở trên khi cuộn trang)
  • Container: Max-width 1200px, căn giữa trang
  • Spacing: Dùng gap, padding, margin nhất quán (8px, 16px, 24px...)

3.1.3.4. TÍCH HỢP VIEW VÀ TEMPLATE

Các View (hàm xử lý) được tạo để:

A) TRANG CHỦ (home_view)
   
   def home(request):
       featured_post = Post.objects.filter(
           status='published'
       ).order_by('-views_count').first()
       
       latest_posts = Post.objects.filter(
           status='published'
       ).order_by('-published_at')[:10]
       
       trending_posts = Post.objects.filter(
           status='published'
       ).order_by('-views_count')[:5]
       
       context = {
           'featured_post': featured_post,
           'latest_posts': latest_posts,
           'trending_posts': trending_posts,
       }
       return render(request, 'trang_chu.html', context)

B) CHI TIẾT BÀI VIẾT (post_detail_view)
   - Tăng views_count mỗi khi người dùng xem
   - Lọc bình luận đã được duyệt
   - Lấy bài viết liên quan (cùng category)

C) DANH MỤC (category_view)
   - Lọc bài viết theo category_id
   - Hỗ trợ phân trang (Paginator)

D) TÌM KIẾM (search_view)
   - Sử dụng Q objects để tìm trên title hoặc content:
   
   from django.db.models import Q
   
   posts = Post.objects.filter(
       Q(title__icontains=query) | Q(content__icontains=query),
       status='published'
   )

3.1.3.5. TẬP TIN STATIC FILES

A) CSS (style.css)

   • Layout chính: Flexbox cho header/footer/main content
   • Component styles: .news-card, .featured-card, .btn, etc.
   • Media queries: Responsive breakpoints

B) JAVASCRIPT (trang_chu.js)
   • Form validation: Kiểm tra input trước khi gửi
   • Smooth scrolling: Cuộn mượt mà khi click link
   • AJAX (nếu cần): Load bài viết mà không reload trang

3.1.4. KẾT QUẢ XÂY DỰNG

3.1.4.1. CƠ SỞ DỮ LIỆU

✅ HOÀN THÀNH:
  • Khởi tạo 7 bảng chính: User, Category, Tag, Post, Comment, Advertisement, 
    Post_Tag
  • Các ràng buộc dữ liệu: Primary Key, Foreign Key, Unique constraints được 
    định nghĩa rõ
  • Hỗ trợ truy vấn ORM hiệu quả: Filter, Order by, Join tự động
  • Data integrity: Khi xóa một Category, tất cả Post của nó cũng bị xóa (CASCADE)

3.1.4.2. GIAO DIỆN NGƯỜI DÙNG

✅ HOÀN THÀNH:
  • Trang chủ: Hiển thị bài viết nổi bật, mới nhất, trending
  • Chi tiết bài viết: Nội dung đầy đủ, bình luận, bài liên quan
  • Danh mục: Lọc bài viết theo chủ đề
  • Tìm kiếm: Tìm kiếm toàn văn
  • Đăng nhập / Đăng ký: Quản lý tài khoản người dùng
  • Bình luận: Gửi và duyệt bình luận
  • Responsive: Hoạt động tốt trên desktop, tablet, mobile

3.1.4.3. CHỨC NĂNG BỔ SUNG

✅ THÊM ĐƯỢC CÀI ĐẶT:
  • Django Admin panel: Cho phép quản trị viên CRUD các Model từ giao diện web
  • User Authentication: Đăng nhập, đăng xuất, session management
  • Slug generation: Tự động tạo URL-friendly identifier từ tiêu đề
  • Pagination: Phân trang danh sách bài viết (nếu quá 10 bài)
  • Search functionality: Tìm kiếm bài viết

3.1.4.4. THỐNG KÊ

Cơ sở dữ liệu:
  • Tổng số bảng: 14+ (bao gồm bảng Django mặc định)
  • Quan hệ dữ liệu: 8 Foreign Keys, 1 Many-to-Many relationship
  • Constraint: 5+ Unique, Multiple NOT NULL

Giao diện:
  • Tổng số Template: 9 file HTML
  • Tỷ lệ Responsive: 100% (hỗ trợ 3+ breakpoints)
  • Tối ưu hiệu suất: CSS/JS minimized, ảnh compressed

Code Quality:
  • Comment hóa chi tiết trong models.py, views.py
  • Cấu trúc folder rõ ràng (templates/, static/, migrations/)
  • Sử dụng Django best practices (ForeignKey relationships, choice fields...)
