3.2.1. GIỚI THIỆU

Giai đoạn này mô tả quá trình triển khai các mô-đun chính của website Tin tức & Blog, 
tập trung vào thực tiễn xây dựng code, xử lý logic nghiệp vụ, và tích hợp các chức năng 
hoạt động. Khác với Chương 2 (thiết kế lý thuyết), phần này trình bày cách các mô-đun 
được code và triển khai để giải quyết các yêu cầu thực tế.

Các mô-đun chính được phát triển bao gồm:
  1. Mô-đun Xác thực (Authentication Module) - Đăng ký, đăng nhập, đăng xuất
  2. Mô-đun Quản lý nội dung (Content Management Module) - CRUD cho Post, Category, Tag
  3. Mô-đun Bình luận (Comment Module) - Gửi, hiển thị, duyệt bình luận
  4. Mô-đun Tìm kiếm (Search Module) - Tìm kiếm bài viết theo từ khóa
  5. Mô-đun Danh mục (Category Module) - Lọc bài viết theo chủ đề
  6. Mô-đun Quản lý giao diện Admin (Admin Dashboard Module) - Django Admin Customization

3.2.2.3. VIEW ĐĂNG KÝ

File: views.py

  def register_view(request):
      """Đăng ký tài khoản"""
      if request.method == 'POST':
          form = RegisterForm(request.POST)
          if form.is_valid():
              # Tạo user mới
              user = form.save()
              
              # Tự động đăng nhập sau khi đăng ký
              login(request, user)
              
              # Redirect về trang chủ
              return redirect('webtintuc:index')
      else:
          form = RegisterForm()
      
      context = {
          **get_base_context(),
          'form': form,
      }
      return render(request, 'register.html', context)

Quy trình:
  1. Kiểm tra phương thức HTTP (POST = form submission, GET = hiển thị form)
  2. Nếu POST: Xác thực form, nếu hợp lệ tạo user và tự động đăng nhập
  3. Nếu GET: Hiển thị form rỗng cho người dùng nhập
  4. Lỗi xác thực (email tồn tại, password không khớp) tự động hiển thị trên form

3.2.2.4. VIEW ĐĂNG NHẬP

  def login_view(request):
      """Đăng nhập"""
      if request.method == 'POST':
          form = LoginForm(request.POST)
          if form.is_valid():
              username = form.cleaned_data['username']
              password = form.cleaned_data['password']
              
              # Xác thực credentials
              user = authenticate(request, username=username, password=password)
              
              if user is not None:
                  # Đăng nhập thành công
                  login(request, user)
                  
                  # Redirect về trang trước đó hoặc trang chủ
                  return redirect(request.GET.get('next', 'webtintuc:index'))
              else:
                  # Username hoặc password sai
                  form.add_error(None, 'Tên đăng nhập hoặc mật khẩu không đúng')
      else:
          form = LoginForm()
      
      context = {
          **get_base_context(),
          'form': form,
      }
      return render(request, 'login.html', context)

Tính năng đặc biệt:
  ✓ request.GET.get('next', ...) - sau đăng nhập, redirect về trang mà người dùng 
    cố truy cập lúc chưa login
  ✓ form.add_error(None, ...) - Thêm lỗi tổng quát (không liên kết field nào cụ thể)
  ✓ authenticate() - Hàm Django kiểm tra username/password hợp lệ

  @admin.register(Post)
  class PostAdmin(admin.ModelAdmin):
      # Cột hiển thị trong danh sách
      list_display = ('title', 'author', 'category', 'status', 
                      'post_type', 'views_count', 'created_at')
      
      # Bộ lọc
      list_filter = ('status', 'post_type', 'category', 'created_at')
      
      # Tìm kiếm
      search_fields = ('title', 'slug', 'content')
      
      # Widget dropdown cho Many-to-Many (tags)
      filter_horizontal = ('tags',)
      
      # Tự động sinh slug từ title
      prepopulated_fields = {'slug': ('title',)}
      
      # Chia form thành các nhóm (fieldsets)
      fieldsets = (
          ('Thông tin cơ bản', {
              'fields': ('title', 'slug', 'post_type', 'category')
          }),
          ('Nội dung', {
              'fields': ('summary', 'content')
          }),
          ('Tác giả & Thẻ', {
              'fields': ('author', 'tags')
          }),
          ('Đăng bài', {
              'fields': ('status', 'views_count', 'created_at', 'published_at'),
              'classes': ('collapse',)  # Mục này mặc định bị ẩn
          }),
      )
      
      # Không cho phép chỉnh sửa trực tiếp
      readonly_fields = ('created_at', 'views_count')

Tính năng nâng cao:
  ✓ fieldsets - Tổ chức form thành các tab/nhóm hợp lý
  ✓ filter_horizontal - Widget chọn nhiều tag (lên xuống) thay vì select box
  ✓ readonly_fields - Hiển thị nhưng không cho phép chỉnh sửa
  ✓ 'classes': ('collapse',) - Nhóm có thể thu gọn

C) ADMIN BÌNH LUẬN

  @admin.register(Comment)
  class CommentAdmin(admin.ModelAdmin):
      list_display = ('user', 'post', 'is_approved', 'created_at')
      list_filter = ('is_approved', 'created_at')
      search_fields = ('user__username', 'content', 'post__title')
      
      # Custom actions (hành động bulk)
      actions = ['approve_comments', 'disapprove_comments']
      
      def approve_comments(self, request, queryset):
          """Duyệt nhiều bình luận cùng lúc"""
          updated = queryset.update(is_approved=True)
          self.message_user(request, f'{updated} bình luận đã được duyệt.')
      approve_comments.short_description = "Duyệt bình luận đã chọn"
      
      def disapprove_comments(self, request, queryset):
          """Ẩn nhiều bình luận cùng lúc"""
          updated = queryset.update(is_approved=False)
          self.message_user(request, f'{updated} bình luận đã bị ẩn.')
      disapprove_comments.short_description = "Ẩn bình luận đã chọn"

Tính năng đặc biệt:
  ✓ actions - Các hành động bulk có thể chọn nhiều item rồi thực hiện
  ✓ queryset.update() - Cập nhật nhiều record cùng lúc (hiệu quả hơn for loop)
  ✓ self.message_user() - Hiển thị thông báo cho admin sau khi thực hiện action

3.2.4.2. XỬ LÝ BÌNH LUẬN TRONG CHI TIẾT BÀI VIẾT

File: views.py

  def post_detail(request, slug):
      """Chi tiết bài viết"""
      post = get_object_or_404(Post, slug=slug, status='published')
      
      # Tăng lượt xem
      post.views_count += 1
      post.save(update_fields=['views_count'])
  
      # Bài viết cùng danh mục (liên quan)
      related_posts = Post.objects.filter(
          category=post.category,
          status='published'
      ).exclude(id=post.id)[:3]
  
      # Bình luận đã duyệt
      comments = Comment.objects.filter(
          post=post, 
          is_approved=True
      ).order_by('-created_at')
      
      # Xử lý form bình luận
      comment_form = None
      if request.user.is_authenticated:
          if request.method == 'POST' and 'comment' in request.POST:
              comment_form = CommentForm(request.POST)
              if comment_form.is_valid():
                  # Tạo bình luận nhưng chưa lưu
                  comment = comment_form.save(commit=False)
                  comment.post = post
                  comment.user = request.user
                  
                  # Mặc định bình luận phải được duyệt
                  comment.is_approved = False
                  comment.save()
                  
                  return redirect('webtintuc:post_detail', slug=post.slug)
          else:
              comment_form = CommentForm()
  
      context = {
          **get_base_context(),
          'post': post,
          'related_posts': related_posts,
          'comments': comments,
          'comment_form': comment_form,
      }
      return render(request, 'chi-tiet-tin.html', context)

Chi tiết xử lý:

  1. Tăng lượt xem:
     post.views_count += 1
     post.save(update_fields=['views_count'])
     → Chỉ cập nhật field views_count (hiệu quả hơn save toàn bộ)
  
  2. Lọc bình luận:
     Comment.objects.filter(post=post, is_approved=True)
     → Chỉ hiển thị bình luận đã được quản trị viên duyệt
  
  3. Xử lý form bình luận:
     if request.user.is_authenticated:
     → Chỉ cho phép người dùng đã đăng nhập bình luận
     
     if 'comment' in request.POST:
     → Kiểm tra xem có form bình luận được gửi (không phải form khác)
     
     comment = comment_form.save(commit=False)
     → commit=False: Không lưu ngay, để set thêm dữ liệu
     → Set post và user tự động
     → Set is_approved=False: Bình luận cần duyệt trước khi hiển thị

3.2.5. MÔ-ĐUN TÌM KIẾM (SEARCH MODULE)

3.2.5.1. VIEW TÌM KIẾM

File: views.py

  def search_posts(request):
      """Tìm kiếm bài viết"""
      query = request.GET.get('q', '').strip()
      
      if query:
          # Tìm kiếm bài viết có title chứa từ khóa
          posts = Post.objects.filter(
              status='published',
              title__icontains=query
          ).order_by('-published_at')
      else:
          posts = []
  
      context = {
          **get_base_context(),
          'query': query,
          'posts': posts,
      }
      return render(request, 'search_results.html', context)

Tính năng:
  ✓ request.GET.get('q') - Lấy query string từ URL (?q=keyword)
  ✓ .strip() - Xoá khoảng trắng đầu/cuối
  ✓ title__icontains - Tìm kiếm case-insensitive (không phân biệt hoa thường)
  ✓ Chỉ tìm bài đã published
  ✓ Sắp xếp theo published_at (mới nhất trước)

Cấu hình URL:

  urlpatterns = [
      path('tim-kiem/', views.search_posts, name='search'),
  ]

Cách sử dụng:
  ✓ URL: /tim-kiem/?q=django
  ✓ Template: <form action="{% url 'webtintuc:search' %}" method="get">
              <input name="q"></form>

3.2.10. BẢO MẬT VÀ TỐI ƯU

┌─────────────────────────┬──────────────────────────────────────────────┐
│ VẤN ĐỀ                  │ GIẢI PHÁP TRIỂN KHAI                         │
├─────────────────────────┼──────────────────────────────────────────────┤
│ SQL Injection           │ Dùng Django ORM (không string concatenation).│
│                         │ Các query như Post.objects.filter(            │
│                         │ title__icontains=query) tự động escape       │
├─────────────────────────┼──────────────────────────────────────────────┤
│ CSRF Protection         │ Form Django tự động add {% csrf_token %};    │
│                         │ middleware CsrfViewMiddleware bật mặc định   │
├─────────────────────────┼──────────────────────────────────────────────┤
│ Xác thực bình luận      │ Chỉ hiển thị bình luận is_approved=True;    │
│                         │ bình luận mới set is_approved=False          │
├─────────────────────────┼──────────────────────────────────────────────┤
│ N+1 Query Problem       │ Dùng select_related(), prefetch_related()    │
│                         │ (nếu cần tối ưu thêm)                        │
├─────────────────────────┼──────────────────────────────────────────────┤
│ Views không được cache  │ Mỗi lần load trang, views_count tăng 1      │

  ✅ Lọc bài theo danh mục → danh sách đúng
